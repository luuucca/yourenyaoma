# Product

## Register

product

## Users

**Primary**: 海外华人邻里——常驻德国/欧洲的留学生、搬家/回国清仓者、新到本地需要置办家具家电的家庭、本地华人家庭、想买便宜二手好物的人。

**Context**: 多数用户已经在小红书 / 微信群里活跃，习惯中文交流，对"闲鱼那种纯交易场景"无感，更想要的是一个**邻里感**的同城社区——能就近自取、能找到靠谱中文服务师傅、能在异国找到一起玩的搭子。

**Job to be done**:
1. 把家里闲置的东西低成本流通给同城邻居（卖、送、整屋打包）
2. 在异国找到能用中文沟通的认证服务（翻译、搬家、维修、装修、月嫂、中医）
3. 在异国找到能一起玩 / 一起学习 / 拼车的同城搭子

## Product Purpose

「有人要吗」是面向海外华人的**邻里社区平台**，一个 app 解决三件事：闲置二手 / 靠谱师傅 / 同城搭子。slogan：「你不要的，正好有人要」。

**Success**:
- 同城自取交易完成率（不只是浏览）
- 站内 IM 留痕沟通比例（vs 私加微信跳出）
- 双向评价沉淀的认证师傅 / 搭子局复购率
- "本周活动数" 和 "整屋打包数" 这类邻里粘性指标

**第一版不做**：站内支付、抽佣、身份证/手机短信验证、App。先做信息发布 + 浏览 + 搜索 + 联系 + 人工审核。变现远景：商品置顶、首页推荐、搬家甩卖套餐、师傅广告位。

## Brand Personality

**3 词**：邻里 · 温柔 · 务实

**Voice**:
- 像跟邻居说话，不像客服 / 不像广告人
- 中文为主，必要时德语 / 英语穿插（如 Gratis · Take it），不强行翻译
- 不鸡汤、不喊口号；"漂泊的日子也要有邻居一起玩" 这种带点诗意但落地的表达 ok
- 数据 + 邻居语录，不靠形容词堆叠

**Emotional goals**:
- 「这里有我同城的人」的安全感
- 「不用从零打听」的省事感
- 「举手之劳能帮到邻居」的温暖感

## Anti-references

**绝对不要**：
- 闲鱼/转转：广告位密、ICON 堆砌、刺激消费的红色标签、推荐流强奸用户
- 淘宝/拼多多：营销弹窗、补贴红包、土味闪闪发光按钮
- 传统二手网站（craigslist / leboncoin / Kleinanzeigen）：信息密度大、视觉冗杂、缺乏温度
- 通用 AI Saas 模板：紫蓝渐变、玻璃拟态、3 列 metric "10x faster"、Inter + Geist 字体堆栈

**也不要**：
- 强行出现"维也纳/柏林"字样（品牌不绑定单一城市；区域字段用 `Berlin Mitte · 1.2km` 这种自然标注）
- 用 emoji 替代设计（emoji 是点缀，不是核心 IA）
- 用紫色或蓝色品牌色（黄黑米才是品牌底色）

## Design Principles

1. **邻里温度，不是 marketplace 冷感**
   每个 section 都要让人感觉"这是有人住在里面的社区"。用「邻居刚上架 · 3km 以内」「漂泊的日子也要有邻居一起玩」这种带温度的眉标，而不是「最新发布」「热门活动」这种功能化措辞。

2. **同城见面是终点，不是 GMV**
   产品决策上：站内 IM 留痕 > 私加微信跳出；公共场所自取 > 邮寄；双向评价沉淀 > 一次性成交。视觉上也要持续强化这条线索（自取 pill、地理距离、评价勋章）。

3. **文字直接，不长篇大论**
   按钮就两个字（聊一聊、我要、加入）。section 描述一行讲透（"3 km 以内 · 实时刷新 · 站内 IM 聊价 · 公共场所自取"），不写"我们致力于"。

4. **明牌交易安全**
   不藏规则。"3 步安心交易"section 黑底反白单独立项，不写在 Footer 角落。每个师傅卡片明显有「✓ 已认证」+ 资质 tag。

5. **简洁但不冷淡 — 一种克制的活泼**
   黄黑米色 + 衬线大标题 + 1px 细线（克制）；同时手绘 PNG 插画 + Caveat 风手写贴纸 + 黄色高光（活泼）。两者要并存，不要往任何一端跑偏。

## Promotions / Growth

### 冷启动抽奖（待实现 — 任务 #42）

**目的**：吸引海外华人邻里完成「注册 + 真实发布」这条最关键的转化路径。

**机制**：
- 奖品：€200 MediaMarkt 代金券（1 名）
- 资格：① 邮箱已验证 ② 至少有 1 条 `status='published'` 的 listing ③ 注册时间 ≥ 抽奖活动启动日
- 抽签方式：random.org signed API（returns a verifiable signed result URL）
- 公示：中奖者昵称 + 邮箱前 3 位脱敏 + signed receipt 链接挂在 `/giveaway/rules`
- 频率：暂定每月 1 次，季节性主题（搬家季 / 开学季 / 圣诞季）可加码

**marquee 文案**（待加入 12 条循环）：
> 🎁 新邻居福利 · 注册并成功发布闲置即可参与 €200 MediaMarkt 代金券抽奖

**合规要点**（德国 / 欧盟）：
- 公开抽奖规则（中文 + 德文双语推荐）：参与条件、截止日期、不可转让、与产品无关、个人数据用途
- 不收取参与费用 / 不强制购买（抽奖纯粹 reward，不是博彩）
- 中奖者邮箱明示用于通知 + 兑奖，不可二次营销
- 保留 random.org signed result URL 作为公平证据

**数据模型**：
- 新建 `giveaways` 表（id / title / prize / starts_at / ends_at / status / created_by）
- 新建 `giveaway_draws` 表（id / giveaway_id / winner_user_id / random_org_url / drawn_at / drawn_by）
- 资格查询通过 view 或 RPC 函数封装，admin 后台直接调用

## Accessibility & Inclusion

- **WCAG AA** 作为最低标准（contrast 4.5:1 normal / 3:1 large）
- **中文用户为主**：所有 UI 文案中文为先，必要时双语；font-feature-settings: 'palt' 优化中文标点
- **键盘可达**：所有交互元素 focus-visible 黄色描边
- **触摸热区**：移动端 ≥ 44×44 px
- **降低动效偏好**：`prefers-reduced-motion: reduce` 全局生效（marquee 暂停、过渡降到 0.01ms）
- **色盲**：黄/黑/米作为主色对所有色盲类型都安全；状态信息不只靠颜色（如「✓ 已认证」有图标 + 文字 + 颜色三重提示）
- **屏幕阅读器**：装饰元素 `aria-hidden`；图标按钮强制 `aria-label`；marquee `role="marquee"` + `aria-live="off"`
